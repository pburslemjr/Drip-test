1 Design Architecture

1.1 Subject Vehicle

The collision warning system is a part of a vehicle control system which is a part of whole vehicle system. It's simplified architecture is summarized in SubjectVehicle module - Fig. 1.1-A.

Figure 1.1-A: Subject Vehicle - subjectVehicle

1.2 Obstacle

For testing purposes obstacle component (Fig. 1.2-A) has been introduced. It represents any obstacle which can subject vehicle meet - another vehicle or not moving object.

Figure 1.2-A: Obstacle - obstacle

1.3 System of Two Objects

Both components (subjectVehicle and obstacle) are contained in one system (CWSsystem) in which they interact together - Fig. 1.3-A. This system is used as the testing environment.

Figure 1.3-A: Testing environment - CWSsystem

1.5 Low Level Requirements

Two lovel level requirements were defined - Fig. 1.5-A. First is a goal which was be used of parameter optimization. Second is a constraint and was used to verify the design. It consists of a pre-constraint (existence expression) and constraint (universality expression).

Figure 1.5-A: Low level requirements - Requirement1

1.6 Test Cases

Three test cases were defined. All of them defined for System of Two Objects. However, they differ only in the setup part. The first test case TestCase1 corresponds to High Cost High Range Sensor and Tactile Warning System (Fig. 1.6-A). It is reflected in two parameters - detectionRange and driverReactionTime. The second test case TestCase2 corresponds to Low Cost Low Range Sensor and Tactile Warning System (Fig. 1.6-B). In the third test case TestCase3 (Fig. 1.6-C) these parameters were not set directly, they were optimized using simulations (Fig. 1.6-D).
